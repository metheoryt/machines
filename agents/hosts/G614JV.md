### WSL
- Personal repos live under `~/gh/` inside WSL. **Ubuntu-24.04** = restored personal env (metheoryt key as `id_ed25519`, repos already present). **Ubuntu-26.04** = set up 2026-07 with BOTH keys mirrored from Windows (`id_rsa`=metheoryt default, `id_ed25519`=cyphy671 via `githubcyphy` alias) + matching `~/.ssh/config`; holds `~/gh/{qaz-code,airdrome,vasya}`.
- To run qaz-code's Docker stack inside a WSL distro, enable that distro in Docker Desktop → Settings → Resources → WSL Integration (per-distro toggle; 26.04 was OFF by default).

### Tooling gotcha — PowerShell tool + `wsl … bash -c`
- Invoking `wsl -d <distro> -u me -- bash -c '…$var…'` **through the PowerShell tool loses shell variables** (they expand empty) and cwd falls back to the `/mnt/c` Windows dir. Fix: write the script to a file with LF endings (use the Bash/Git-Bash tool) and run it via `wsl -d <distro> -u me -- bash /mnt/c/…/script.sh`.
