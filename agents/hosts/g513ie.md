## Notes

- **Russian-locale Windows breaks English ACL group names.** `icacls` / PowerShell
  ACL grants that reference `Administrators` or `SYSTEM` by English name silently fail
  ("no mapping between account names and security IDs was done") without raising a
  script error unless the exit code is checked — grant by **well-known SID** instead
  (`*S-1-5-32-544` = Administrators, `*S-1-5-18` = SYSTEM).
- **Git "dubious ownership" from Administrators-group-owned dirs.** A repo dir owned by
  `BUILTIN\Администраторы` (Administrators) instead of the user trips git's safety
  check and blocks `git pull`/etc. Quick unblock: `git config --global --add
  safe.directory <path>`; the real fix is `icacls <path> /setowner "<user>" /T /C`
  (NOT `takeown` alone, which assigns ownership to the elevated admin identity, not
  the actual user).
- **Scheduled-task console flash / Docker Desktop coupling.** An `Interactive`-logon
  Windows Scheduled Task flashes a `conhost` window on every run — fix by wrapping the
  action in `conhost.exe --headless`, NOT by switching to S4U ("run whether logged on
  or not"). Docker Desktop's daemon here is reachable only from the interactively
  logged-in user's session (per-user pipe/context), so an S4U or SYSTEM task would
  lose `docker` access.
- **sshd only reads `administrators_authorized_keys` for admin accounts.** `methe` is
  an Administrator, so Windows sshd ignores per-user `~/.ssh/authorized_keys` by design
  and uses ONLY `C:\ProgramData\ssh\administrators_authorized_keys`. That file must
  have a tight ACL (SYSTEM + Administrators only, no Authenticated Users read —
  `icacls /inheritance:r`) or sshd silently rejects it and falls back to password auth.
  Apply changes with `Restart-Service sshd` (no reboot needed).
- **Keep PowerShell 7 installed via MSI, not the Store.** sshd's `DefaultShell`
  registry value points at `C:\Program Files\PowerShell\7\pwsh.exe`; a Store/MSIX
  install lives at a versioned WindowsApps path, so that fixed path goes missing and
  sshd rejects `methe` as an invalid user (killing BOTH key and password auth). Install
  via `https://aka.ms/install-powershell.ps1 -UseMSI`. `winget install
  Microsoft.PowerShell` fails here with "system configuration does not support
  installation" (a winget-manifest applicability quirk, not a real OS block).
- **Headless git push uses a dedicated account SSH key.** GCM-based push fails headless
  (no tty / can't write wincredman); the working setup is an account key
  `~/.ssh/id_ed25519_gh` (no passphrase) with `~/.ssh/config` (`IdentitiesOnly yes` for
  github.com) and SSH remotes. NOTE: the pre-existing `~/.ssh/id_ed25519` is a
  **telegrind-scoped deploy key** ("Hi metheoryt/telegrind!"), not an account key — it
  can't push to other repos.
- **The Windows `machines` checkout carries persistent uncommitted drift** on ~10
  tracked files (settings / `.gitattributes` / `.gitignore` / `.mcp.json` / `AGENTS.md`
  / etc.), largely CRLF or machine-local — it will eventually collide with `git pull`
  (and with `/ship`'s dirty-check). Reconcile before a pull that touches those paths.
- **Docker over SSH: remove `credsStore`.** `credsStore: "desktop"` in
  `~/.docker/config.json` makes every pull fail with "A specified logon session does
  not exist" (no interactive logon session for Credential Manager over SSH). Removing
  `credsStore` fixes it — this box only pulls public images.
- **WSL2/Docker Desktop mount tree can wedge** with `mkdir /run/desktop/mnt/host/e:
  file exists` for a drive (e.g. E:) even when the host drive is healthy. Reliable fix:
  `wsl --shutdown` (rebuilds `/run/desktop/mnt/host/*`; Docker Desktop auto-restarts
  its engine, no elevation) — but it briefly bounces every container.
- **Docker from Git Bash needs `MSYS_NO_PATHCONV=1`.** `docker exec`/`docker run` calls
  from Git Bash (MINGW64) mangle container paths unless prefixed with
  `MSYS_NO_PATHCONV=1`.
- **kb-refresh tooling caveats here:** no `jq` and no ssh fleet aliases (only
  `github.com` in `~/.ssh/config`), so `fleet-gather.sh` can't run — cross-machine kb
  gather is local-only on this box. The skill's `distill.py` needs `PYTHONUTF8=1` or it
  hits a cp1251 `UnicodeEncodeError` on the `→` character when writing digests.
