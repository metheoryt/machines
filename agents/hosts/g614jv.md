### Tooling gotcha — PowerShell tool + `wsl … bash -c`
- Invoking `wsl -d <distro> -u me -- bash -c '…$var…'` **through the PowerShell tool loses shell variables** (they expand empty) and cwd falls back to the `/mnt/c` Windows dir. Fix: write the script to a file with LF endings (use the Bash/Git-Bash tool) and run it via `wsl -d <distro> -u me -- bash /mnt/c/…/script.sh`.
